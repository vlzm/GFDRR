# Post-Task Process

Read this after completing a code task.

## Update PROJECT_STATE.md

Check if `PROJECT_STATE.md` should be updated:
- Roadmap item was completed or started
- New architecture decisions were made
- File map changed (new directories, renamed files)
- Current phase status changed

Do NOT update just because code was edited — only if the *project state* actually changed.

## Validate Against Data Model
Verify that changes are consistent with the data model invariants in `docs/design/graph_data_model.md`.
