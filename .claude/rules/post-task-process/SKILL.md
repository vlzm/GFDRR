# Post-Task Process

Read this after completing a code task.

## Update PROJECT_STATE.md

Check if `PROJECT_STATE.md` should be updated:
- Cleanup progress changed (modules removed, simplified)
- New issues discovered during cleanup

Do NOT update just because code was edited — only if the *project state* actually changed.

## Validate Against Canonical Scenario

After any code change, verify that `notebooks/canonical_scenario.ipynb` still runs correctly. The canonical scenario is the source of truth for what must work.
